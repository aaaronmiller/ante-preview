## 📋 Master Plan: PRD (What & Why)

This PRD defines the feature set to transform Ante into a best-in-class AI coding agent.

### 🎯 Core Goal
To elevate Ante from a powerful local-model agent into a **fully extensible, autonomous, and collaborative development platform**. This involves integrating external capabilities, implementing a best-in-class hook system, and introducing cutting-edge features for memory, orchestration, and multi-agent communication.

### 🗺️ Feature Overview and Prioritization

| Priority | Feature Category | Core Capabilities | Why Integrate into Ante? | Ante Mechanism |
| :--- | :--- | :--- | :--- | :--- |
| **P0 - Foundational** | **Hook System** | Event-driven automation that intercepts and modifies agent behavior at every lifecycle stage. | This is the single most impactful feature for user trust and customization. It enables security policies, workflow automation, and context injection—all without modifying Ante's core. It is a must-have for enterprise adoption and directly competes with Claude Code and Cursor. | New internal module: `Hook Manager` + `Event Dispatcher` |
| **P0 - Foundational** | **MCP Ecosystem Integration** | Native, token-efficient gateway to the entire MCP server ecosystem for databases, browsers, APIs, and more. (`pi-mcp-adapter`, `pi-web-search`) | This makes Ante a universal agent, breaking out of its sandbox to interact with real-world tools and data. It's the primary mechanism for extending capabilities without bloating the core. | MCP Server configuration in `settings.json` |
| **P1 - Power** | **Multi-Agent Orchestration** | A framework for decomposing complex tasks, spawning specialized sub-agents, and executing them in parallel or asynchronously. (`pi-subagents`, `pi-astro`) | Enables Ante to tackle complex, multi-step problems that require specialized focus. This moves Ante from a single-agent tool to a team manager, dramatically improving outcomes on large-scale tasks. | Built-in Sub-Agent system + Task Decomposition Engine |
| **P1 - Power** | **Persistent Memory** | A long-term, semantic memory system that learns from past interactions, remembers project-specific decisions, and recalls relevant context across sessions. | Addresses the "goldfish problem" where agents forget everything between sessions. This is a critical differentiator for long-running projects, reducing repetitive instructions and improving context-aware decision-making. | MCP Server for Memory + SessionStart/End Hooks |
| **P2 - Usability** | **Skills & UI Enhancements** | Terminal-based Mermaid diagram rendering, persistent to-do list management, and structured task planning. (`pi-mermaid`, `rpiv-todo`) | Improves the agent's ability to communicate complex plans and manage multi-step tasks visibly, which is essential for user oversight of autonomous work. | Skills via `SKILL.md` files |
| **P2 - Usability** | **Dynamic Model Switching** | An intelligent router that selects the optimal model (local or API) for each task based on complexity, cost, latency, and privacy requirements. | This enables "smart cost savings" by using cheap, fast models for simple tasks and powerful, expensive models for hard ones—all transparently to the user. It's a critical feature given Ante's local-model focus. | New module: `Model Router` |
| **P3 - Innovation** | **Inter-Agent Communication Protocol** | A standardized protocol (like A2A) for direct, real-time communication between separate Ante sessions or other AI agents. (`pi-intercom`) | Allows for complex, multi-session workflows where different contexts (e.g., research vs. execution) need to coordinate, and enables future "agent team" topologies. | MCP Server for a local message broker |
| **P3 - Innovation** | **Human-in-the-Loop (HITL) Approval** | A framework that pauses agent execution for sensitive tool calls until a human explicitly approves, rejects, or modifies the action. | This is essential for building trust and ensuring safety in autonomous operations, particularly in production environments. It's a direct complement to the hook system. | Extension of the Hook System (`PermissionRequest` event) |
